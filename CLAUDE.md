## Development Guidelines

### Git Conventions

- **Commit messages must never mention AI, Claude, or any AI assistant.** No `Co-Authored-By` AI lines.
- Write commit messages in imperative mood (e.g., "Add feature" not "Added feature").
- Keep the first line under 72 characters.
- Use conventional commit prefixes when appropriate: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`.

### Code Style

- Follow existing patterns and conventions in the codebase.
- Prefer clarity over cleverness.

### Testing

- Run tests before committing when a test suite exists.
